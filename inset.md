# 차량 밑 삽입(Insertion) 알고리즘

두 협동 로봇(front / rear)이 target_vehicle **밑으로 부딪히지 않고 진입**하여,
아두이노 HC-SR04급 초음파(단일 거리값)만으로 앞/뒤 바퀴 축을 찾아 정렬하는
알고리즘 설명. 구현: `map_astar_planner.py` + `gripper_controller.py`

---

## 1. 왜 이렇게 하는가 — 실패했던 방식

처음에는 A* 플래너가 **차량 중심 좌표**를 목표로 로봇 쌍을 안내했다.
그 결과:

- 경로가 대각선(측면)에서 차량 중심으로 파고듦 → 로봇이 차량
  바퀴·차체 모서리에 **부딪혀 차량을 밀어버림**
- CCTV는 밀려나는 차량을 계속 추적해 재계획 → 로봇이 차량을
  계속 밀고 다니는 악순환 (실측: 차량이 (0.63, 0.60) → (0.55, 0.04)까지 밀림)

**교훈: 거친 경로계획(격자 0.2 m, 도착오차 ~4 cm)으로는 차량 밑
진입이 불가능하다. 좌우 여유가 4.5 cm뿐이기 때문이다.**
→ "밖에서 정렬 후 저속 직진 삽입"으로 역할을 분리했다.

## 2. 치수와 여유 (기하 조건)

| 항목 | 값 | 비고 |
|---|---|---|
| 차량 차체 | 0.45 × 0.34 × 0.05 m, 바닥 z = 0.132 | 빨강 (CCTV 감지용) |
| 차량 바퀴 | r 0.05, 폭 0.03, 중심 (±0.125, ±0.17) | 축간거리 0.25 m |
| 바퀴 안쪽면 간격 | ±0.155 → 0.31 m | 통과 폭 |
| 로봇 본체 | 0.20 × 0.20 × 0.08, 상면 z = 0.09 | 밑 여유 4.2 cm |
| 센서 마스트 상단 | z = 0.11 | 차체 바닥과 2.2 cm 여유 |
| 로봇+접힌 그리퍼 폭 | ±0.11 | **좌우 여유 4.5 cm** ← 핵심 제약 |
| 로봇 간격 | 0.25 m | **차량 축간거리와 동일** → 두 로봇이 동시에 자기 축 도달 |
| 초음파 | 옆면 y ±0.10, ±y 방향, z = 0.05 (바퀴 중심 높이) | HC-SR04 대역 |

초음파는 gpu_lidar 5-ray(빔폭 15°)로 흉내내되 **min(ranges) 하나만
단일 거리값으로 사용**한다. 각도 정보는 쓰지 않는다 — 실제 아두이노
초음파와 동일한 정보량 유지.

## 3. 단계별 알고리즘

### 3-1. 정렬점(standoff)까지 경로계획 — `map_astar_planner.py`

```
목표 = (차량중심 x + 0.60, 차량중심 y)     # approach_standoff = 0.60 m
```

- 플래너/강체동기 제어(sim_rigid_body_sync)는 **차량 밖 정렬점까지만** 담당.
  차량 밑으로는 절대 계획하지 않는다.
- 정렬점은 차량 +x쪽 **축선상**에 있으므로, 도착하면 로봇 쌍이
  차량 진입 방향(−x)과 자세(x축 정렬, yaw ≈ 0)를 자연히 갖춘다.
- 도착 판정: `/sync/goal_reached` + 쌍 중심이 정렬점 반경 0.25 m 이내.

### 3-2. 제어권 인계

- gripper_controller가 `/sync/enable = False` 발행
  → sim_rigid_body_sync는 즉시 cmd_vel 발행 중지.
- 이후 삽입·정렬·파지는 gripper_controller가 두 로봇을 직접 제어.

### 3-3. INSERT — 저속 직진 삽입 + 초음파 에지 검출

두 로봇에 동일한 world −x 속도(0.025 m/s)를 주되, 매 틱 로봇마다:

```
vy = clamp( 1.2 × (차량축선 y − 로봇 y), ±0.02 )   # 축선 y 서보
wz = clamp( −1.5 × yaw오차, ±0.15 )                # x축 정렬 유지
   (yaw오차는 0과 π 중 가까운 쪽 기준 — 뒤집힌 자세 허용)
(vx, vy)를 로봇 yaw로 body frame 회전 후 cmd_vel 발행
```

- **좌우 여유가 4.5 cm뿐이므로 이 y-서보/yaw-유지가 충돌 방지의 핵심.**
  20 Hz 초음파 + 0.025 m/s → 샘플당 이동 ~1.3 mm.
- 삽입하며 각 로봇이 자기 초음파 좌/우 각각으로 **"값이 바뀌는 지점"(에지)** 을 기록:

```
에코 = (단일 거리값 < 0.10 m)          # 바퀴 안쪽면까지 ~5.5 cm
에코 시작 x = enter, 에코 종료 x = exit
바퀴 중심 = (enter + exit) / 2          # 에코 구간의 중점
좌/우 중심의 평균 = 그 로봇의 바퀴 축 x
```

- **x-게이트**: rear 로봇은 뒤축에 닿기 전에 앞바퀴를 먼저 지나며
  에코를 받는다. 이를 걸러내기 위해
  - front 로봇: `x > 차량중심 x` 구간의 에코만 인정 (앞 축)
  - rear 로봇: `x < 차량중심 x` 구간의 에코만 인정 (뒤 축)
- **폴백**: 예상 축 위치(차량중심 ±0.125)를 12 cm 지나도록 에코가
  없으면 CCTV 좌표 기반 예상값으로 대체하고 경고.
- 로봇 간격(0.25 m) = 축간거리이므로 두 로봇이 **거의 동시에**
  자기 축에 도달한다. 양쪽 축이 모두 확정되면 정지 → CENTER.

### 3-4. CENTER — 개별 P-제어 정렬

각 로봇을 자기 바퀴 축 x로 독립 정렬:

```
v = clamp(1.5 × dx, ±0.012),  |v| ≥ 0.004 (최소속도),  |dx| < tol이면 정지
```

y-서보/yaw-유지는 계속 적용. 양쪽 모두 tol 이내면 GRIP.

### 3-5. GRIP → ATTACH (이후 단계, 설계)

- 그리퍼 4개(로봇 옆면 y ±0.10 장착)를 180°(접힘, x방향)에서
  90°(전개, ±y방향)로 회전 — 바퀴 **바깥쪽**을 케이지처럼 감싸는 배치
- DetachableJoint attach = 리프트 대역 → `/robot/lifted = True`
- 플래너가 to_goal 단계로 전환, `/sync/enable = True`로 운반 주행 재개

## 4. 파라미터 (gripper_controller)

| 파라미터 | 기본값 | 의미 |
|---|---|---|
| `approach_standoff` | 0.60 | 정렬점 거리 (플래너와 동일해야 함) |
| `insert_speed` | 0.025 | 삽입 직진 속도 [m/s] |
| `echo_threshold` | 0.10 | 초음파 에코 판정 [m] |
| `y_gain` / `max_vy` | 1.2 / 0.02 | 축선 y 서보 |
| `yaw_gain` / `max_wz` | 1.5 / 0.15 | yaw 정렬 유지 |
| `wheel_half_base` | 0.125 | 게이트·폴백 기준 (축간거리/2) |
| `start_tol` | 0.25 | 정렬점 도착 인정 반경 |
| `center_tol` | 0.005 | 축 정렬 허용 오차 |

## 5. 검증 결과 (Gazebo 실측)

- 정렬점 도착: 오차 4.5 cm, yaw −3.7°
- 삽입 중 차량 위치 (0.625, 0.600)에서 **미동 없음** (밀림 문제 해소)
- 초음파 에지 검출 (실제 축: 앞 0.750 / 뒤 0.500):
  - front 축 확정 x = **0.749** (오차 1 mm)
  - rear 축 확정 x = **0.499** (오차 1 mm)
  - rear가 앞바퀴를 지날 때의 에코는 게이트가 정상 차단
- 단일 거리값(HC-SR04 정보량)만으로 mm급 축 검출이 가능한 이유:
  에지 기준이라 거리 정확도가 아닌 **로봇 자기 위치(odom) 정확도**가
  분해능을 결정하기 때문. 실전에서는 엔코더/odom 품질이 한계를 정한다.

## 6. 실전(아두이노) 대응 관계

| 시뮬 | 실전 |
|---|---|
| gpu_lidar 5-ray의 min 하나 | HC-SR04 단일 거리값 (NewPing 등) |
| 에코 < 0.10 m 판정 | 동일 (바퀴 안쪽면까지 ~5.5 cm) |
| odom x 기록 후 에지 중점 | 엔코더 주행거리 기록 후 에지 중점 |
| DetachableJoint attach | 그리퍼 리프트(바퀴 클램핑) |
| `/robot/lifted` | 리프트 완료 → 주행 허가 게이트 |
